## Why Australian English is NOT a Skill

**Australian English should remain in CLAUDE.md as a mandatory project-wide standard.**

**Reasoning**:
- âŒ **Not conditionally triggered** - Should ALWAYS be enforced, not loaded when "relevant"
- âŒ **No trigger point** - User saying "normalize" won't make Claude think "load spelling skill"
- âœ… **Always in context** - Needs to be visible for every response, not progressive disclosure
- âœ… **Project constraint** - Like using Python 3.12 or ap-southeast-2 region

## Benefits of Skills Over Markdown

1. **Progressive disclosure**: Only load what's needed when needed
2. **Composable**: Combine multiple skills (git-workflow + agile-board)
3. **Portable**: Copy skills to other projects/machines
4. **Triggerable**: Claude knows when to use them automatically
5. **Smaller context**: <5k words per skill vs large markdown files

---

## Key Design Principles

From the skill-creator documentation:

1. **Concise is key** - Only add context Claude doesn't already have
2. **Progressive disclosure** - Core in SKILL.md, details in references/
3. **Good descriptions** - Primary trigger mechanism, include "when to use"
4. **Scripts for automation** - Use scripts/ for repetitive tasks
5. **Cross-platform** - Use Python (not bash) for Windows/Mac/Linux compatibility
